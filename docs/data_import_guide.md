# Seastar 数据导入功能操作指南

## 功能概述
Seastar 的数据导入功能主要用于将历史数据（如 CSV 文件）导入到系统中，以支持回测和实盘分析。该功能支持多种数据源类型和导入模式，用户可以通过命令行界面（CLI）进行交互式操作。

## 数据导入的主要步骤

### 1. 设置导入参数
在 CLI 中，用户可以通过以下选项设置导入参数：
- **source path**: 数据源路径，支持文件或目录。
- **source type**: 数据源类型，目前支持 `taobao_csv` 和 `ctp_clean_csv`。
- **import mode**: 导入模式，支持 `incremental`（增量导入）和 `overwrite`（覆盖导入）。
- **config path**: 配置文件路径，默认为 `./bt_plan/import.conf` 或环境变量 `XTRADER_IMPORT_CONFIG`。

### 2. 执行导入
设置完成后，选择 `Execute import` 开始导入。系统会扫描指定路径下的文件，并根据配置执行导入操作。

### 3. 查看导入结果
导入完成后，系统会输出以下信息：
- **导入状态**: 成功或失败。
- **扫描文件数**: 总扫描文件数、成功文件数、跳过文件数、失败文件数。
- **数据行统计**: 新增行数、覆盖行数、跳过行数、无效行数。
- **详细报告**: 每个文件的导入状态和统计信息。

## 配置文件说明
配置文件采用 `key=value` 格式，支持以下字段：
- `data_root`: 数据根目录。
- `report_root`: 导入报告目录。
- `recursive_scan`: 是否递归扫描子目录。
- `taobao_data_domain`: 淘宝数据域。
- `taobao_timeframe`: 淘宝数据时间周期。

示例配置文件路径：`./bt_plan/import.conf`

## 示例操作
以下是一个完整的导入示例：
1. 启动 CLI 并进入导入菜单。
2. 设置数据源路径为 `bin/data/market/live_capture/XSGE/rb2601/1m/`。
3. 设置数据源类型为 `ctp_clean_csv`。
4. 设置导入模式为 `incremental`。
5. 执行导入并查看结果。

## 注意事项
- 确保数据源路径和配置文件路径正确。
- 导入模式选择需根据实际需求决定，`overwrite` 模式会覆盖已有数据。
- 导入报告会生成在指定目录下，可用于后续分析。
