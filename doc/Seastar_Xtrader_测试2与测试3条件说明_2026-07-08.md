# Seastar Xtrader 测试2与测试3条件说明（2026-07-08）

## 1. 背景

本说明用于补充当前最终审核时的两个不确定项：

- 测试2：运行时日志与账户链路是否满足预期
- 测试3：gateway shard / account shard 路由是否符合配置与实际执行路径

已确认事项：

- `build_release.sh` 编译成功，程序可运行
- `build_debug.sh` 编译成功，程序可运行

因此，本文仅定义测试2与测试3的判定条件，不重复编译结论。

---

## 2. 测试2条件说明：运行时日志与账户链路

### 2.1 测试目标

确认程序运行后，账户更新链路未静默失败，且关键初始化日志、错误日志符合预期。

### 2.2 测试前提

- 程序已通过 `build_release.sh` 或 `build_debug.sh` 成功编译
- 程序已能正常启动
- 账户服务初始化流程已进入运行阶段
- 运行环境可观察标准输出或日志文件

### 2.3 必须观察的内容

1. 启动阶段日志
- 应看到账户服务初始化成功信息，或等价的初始化完成信息
- 不应出现启动阶段直接退出、异常崩溃或未初始化即运行的情况

2. 运行阶段日志
- 不应持续出现账户更新失败日志
- 不应出现“账户服务为空却继续吞掉 delta”的静默失败行为
- 若有 delta 被拒绝或失败，应明确打印错误原因与 shard 信息

3. 运行结果一致性
- 若注入成交回报，仓位变化与账户变化应有对应关系
- 若账户服务未初始化，必须明确失败，而不是表面成功

### 2.4 通过标准

测试2通过需满足以下全部条件：

- 启动日志中能确认账户服务初始化成功
- 无连续的账户更新丢弃日志
- 无未初始化却继续运行的情况
- 发生异常时能输出明确错误，而非静默吞掉

### 2.5 失败判据

出现以下任一情况即判定测试2失败：

- 启动后没有账户服务初始化成功日志
- 运行中出现 account delta 被丢弃但程序仍继续运行
- 账户服务为空却没有中止或没有明确报错
- 成交回报已处理，但仓位/账户结果明显不一致

## 3. 测试3条件说明：gateway shard / account shard 路由

### 3.1 测试目标

确认 gateway shard 和 account shard 的配置与实际路由一致，不存在硬编码与配置脱节。

### 3.2 测试前提

- 程序已成功启动
- `runtime_sharded` 已完成初始化
- 可观察 shard 配置值或路由日志
- 最好在可控的配置下运行一次（例如默认配置或显式修改 shard 参数）

### 3.3 必须观察的内容

1. gateway shard 路由
- 发单请求应路由到 gateway shard
- 若 gateway shard 不是 0，行为必须仍然与配置一致
- 不应只依赖固定 shard 0 的硬编码行为

2. account shard 路由
- 成交回报后的账户更新应路由到 account shard
- 如果 account shard 被改成非默认值，实际执行必须跟随配置

3. 配置与执行一致性
- `set_gateway_shard()` 与 `set_account_shard()` 的配置必须真正影响运行路径
- 日志或调试输出应能反映实际路由目标 shard

### 3.4 通过标准

测试3通过需满足以下全部条件：

- gateway shard 的实际执行路径与配置一致
- account shard 的实际执行路径与配置一致
- 不存在“配置已改，但运行仍固定走 shard 0/1”的情况
- 路由变化后，程序仍能正常启动和处理请求

### 3.5 失败判据

出现以下任一情况即判定测试3失败：

- 路由仍硬编码到 shard 0 或固定 shard 1
- 修改 shard 配置后，执行路径没有变化
- 日志显示路由目标与配置不一致
- 路由改动导致程序启动失败或处理失败

## 4. 建议的最小验证顺序

1. 先确认编译与启动成功
2. 再看测试2：账户初始化与运行日志
3. 最后看测试3：shard 路由是否与配置一致

- 测试3结论：通过 / 不通过
