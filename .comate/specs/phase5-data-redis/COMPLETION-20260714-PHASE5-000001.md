## 四、关键设计决策记录

| 决策项 | 方案 | 理由 |
|--------|------|------|
| Redis 值序列化格式 | CSV（逗号分隔，支持引号转义） | 设计文档要求字段可读；JSON 序列化在 web_api_handler 层完成；CSV 格式在 Redis CLI 中直接可读，便于人工排查；序列化/反序列化代码自包含，无第三方依赖 |
| Web API 数据源策略 | Redis 优先 + memory 兜底 | 策略状态优先从 Redis 查询（跨进程可见）；若未连接 Redis 则读取 `strategy_lifecycle_manager` 内存状态作为兜底。数据集和验证结果仅从 Redis 读取（因数据量大不宜内存全量缓存） |
| Keyspace 前缀隔离 | `xtrader:dataset:` / `xtrader:replay:` / `xtrader:strategy:` / `xtrader:event:` / `xtrader:validation:` | 严格按设计文档 5B-1 建议，5 类元信息独立命名空间，便于 `KEYS` 遍历和 TTL 策略差异化 |
| web_api_handler 注册方式 | 独立类 + `future<> start/stop` | 不绑定 httpd 实现细节，后续可灵活挂载到 Seastar `http_server` 或独立 TCP listener；Phase 5 仅要求接口契约可用，不强依赖运行时 web 服务 |
| historical_data_manager 扩展 | 增量添加字段，不改变现有 API 签名 | `dataset_manifest` 新增字段均有默认值，`scan_datasets()`/`find_dataset()`/`load_bars()` 保持向后兼容 |
