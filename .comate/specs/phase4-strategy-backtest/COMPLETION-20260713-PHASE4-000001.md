## 四、关键设计决策记录

| 决策项 | 方案 | 理由 |
|--------|------|------|
| 策略状态枚举 | `created → initialized → warmup → running → stopping → stopped → failed` | 兼容 Seastar sharded 模型 + 策略热切换需求 |
| 策略 owning shard | 单 shard 绑定，跨 shard 仅传不可变事件 | 防止跨策略状态污染（缓解 R4-3） |
| event_bus 分发预算 | 500μs/batch + 256 events/batch 上限 | 防止策略回调阻塞 reactor（缓解 R4-1） |
| 撮合模型 | conservative matching 起步 | 回测偏差可控，后续逐步引入交易所差异参数（缓解 R4-2） |
| 订阅优先级 | priority 降序 fan-out | 确保交易信号优先于风控，风控优先于统计 |
| 交易时段识别 | 按 instrument_id 前缀匹配交易所 | SHFE/INE(09:00-15:00)、CFFEX(09:30-15:15)、DCE/CZCE(21:00-15:00 夜盘跨日) |
